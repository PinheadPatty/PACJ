Problem Definition

Why PACJ exists and what engineering gap it addresses.

System-Level Challenge

Aerial robots and ground robots each have strong but different capabilities. Drones can rapidly navigate complex 3-D spaces and gain overhead visibility, while rovers provide stable, energy-efficient operation for sustained ground tasks. In most field deployments, these systems are operated independently, which limits mission flexibility and increases operator workload.

PACJ addresses this by creating a coupled air-ground architecture where a drone can autonomously attach to, transport, and redeploy a rover. This enables a mission class that neither platform can reliably achieve alone: cooperative search, obstacle bypass, structured inspection, and multi-phase mapping in constrained environments.

🧭

Relocation Across Obstacles

The drone couples with the rover, lifts it vertically, crosses impassable terrain, and redeploys it in a safe zone. This converts vertical access into ground mobility continuity.

🔋

Sustained Mission Endurance

The architecture supports recharge/energy continuity workflows so the rover can remain mission-capable without continuous manual intervention.

🆘

Search and Rescue Utility

The drone provides high-level situational awareness while the rover performs close-range navigation in debris or confined zones. Together they improve coverage, guidance, and survivability outcomes.

🕵️

Discreet Access and Retrieval

In constrained or visibility-sensitive environments, the rover handles low-profile entry while the drone contributes elevation and rapid repositioning when needed.

📊

Data Collection in Difficult Areas

The drone can deploy the rover into otherwise unreachable measurement zones, enabling distributed sensing and continuous relay of mission data to the operator.

🎯

Capstone Objective

Deliver a repeatable, testable, and safe system that demonstrates autonomous coupling, shared situational awareness, and robust teleoperation/autonomy handoff for a drone-rover team.

Concept Design and Operation

Three-part architecture for coupling, autonomy, and operator-supervised mission execution.

🤖

Rover Module

Lightweight rover with matching top geometry for soft coupling, onboard hard docking mechanism, teleoperation capability, and autonomous path planning/localization in known maps.

🚁

Drone Module

Drone with downward camera and ranging support for alignment, adaptive payload-aware control, and safe coupling/transport/redeployment while maintaining controllability.

💻

Operator Application

Laptop app presents synchronized camera streams, mission controls, and mode switching between manual teleoperation and autonomous behaviors.

🛑

Safety Layer

Global emergency-stop behavior interrupts both agents, forcing immediate rover halt and rapid safe drone landing path.

Concept of Operation

  1. Operator monitors drone and rover streams side-by-side from laptop control station.
  2. User can assume manual teleoperation at any time for either platform.
  3. Autonomous command options include takeoff/land, couple-to-rover, exploration, and point navigation.
  4. Rover autonomous options include drive-under-drone, area exploration, and target waypoint navigation.
  5. Drone performs precision vertical alignment and coupling using downward sensing.
  6. Coupled transport executes across obstacles, then drone lowers rover and releases for resumed ground mission.
  7. Emergency-stop remains globally available throughout all modes.

Example Demo Flow

  1. Initialize agents on opposite sides of environmental barrier and validate telemetry/video links.
  2. Rover performs local drive and mapping, requests pickup when blocked by barrier.
  3. Drone aligns, hard-couples, lifts rover, and traverses obstacle zone.
  4. Drone redeploys rover in clear area; rover resumes autonomous path and mapping.
  5. System returns state estimates and mission outputs to operator interface for verification.

System Architecture

Detailed subsystem architecture across agents, sensing, planning, and operator stack.

Agent

Drone

Runs PX4-facing control, payload-aware flight behavior, and autonomous coupling/transfer execution.

Agent

Rover

Runs ground locomotion, coupler actuation, and local mission continuation after drone redeployment.

Operator

Laptop

Hosts teleop/autonomy controls, mission supervision, and real-time video/map visualization.

Middleware

ROS 2 Graph

Shared topics/services connect drone, rover, and laptop for command, telemetry, maps, and sensor streams.

Estimation

TF + VIO Relay

Frame broadcasting and odometry relays align all agents into a shared coordinate context.

Mapping

Depth Pipeline

Compressed depth feeds decompression and SLAM/mapping nodes for shared environment reconstruction.

Manipulation

Coupling Subsystem

Soft/hard coupling mechanics and servo lock control enable pickup, transport, and release cycles.

Safety

E-Stop + Failsafe

Global stop path, link-loss handling, and controlled landing/halt behavior protect hardware and operators.

Inertial + Motion Sensing IMU, wheel encoders, auxiliary short-range sensors Stabilization, motion estimation, and short-range safety checks. Used in state estimation and control loops with visual/depth data.
Perception Processing Depth decompression + SLAM integration Converts compressed depth streams and feeds mapping/localization pipeline. Bridges sensor transport into mapping and operator visualization outputs.
